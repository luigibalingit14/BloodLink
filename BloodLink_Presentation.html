Group: Group 1
SLIDE 03
Overview

Introduction & Features

BloodLink is a Java-based desktop application designed for blood banks to efficiently manage donor records, monitor real-time blood inventory, and generate analytical reports. The system implements role-based access control, ensuring that Administrators and Staff have appropriate levels of access to the system's features. It utilizes MySQL for persistent data storage and Java Swing for its modern, glassmorphism-inspired user interface.
📊
Dashboard Overview
Real-time statistics showing total donors, blood units, critical stock alerts, and recent donations within 7 days.
🔐
Role-Based Access Control
Two-tier user system: Admin has full access, Staff has restricted access to manage day-to-day operations only.
📝
Donor Registration
Register new blood donors with full details — name, age, blood type, contact, address, and donation date.
👥
Donor Management (CRUD)
View, search, update, and delete donor records using a fully-featured table with search and filter capabilities.
🩸
Blood Inventory Tracking
Real-time tracking of all 8 blood types with color-coded CRITICAL alerts when stock drops below 5 units.
📈
Reports & Analytics
Monthly donation trends and blood distribution reports with one-click CSV export functionality.
⚙️
System Settings (Admin)
Admin-exclusive user management: add/delete accounts, change passwords, view system information.

Database Integration

SQL Statements Used

🔐 LOGIN — Verify User Credentials
SELECT * FROM users WHERE username = ? AND password = ? -- ? = PreparedStatement placeholder (SQL Injection Protected)
➕ ADDING NEW CUSTOMER — INSERT Donor
INSERT INTO donors (name, age, blood_group, phone, address, donation_date) VALUES (?, ?, ?, ?, ?, ?)
🔍 SEARCHING CUSTOMER — Filter by Name or Blood Type
SELECT * FROM donors WHERE name LIKE ? OR blood_group LIKE ? ORDER BY id DESC -- % wildcard: matches any characters (e.g., '%O+%' finds all O+ donors)
✏️ UPDATING CUSTOMER INFORMATION — UPDATE Donor
UPDATE donors SET name=?, age=?, blood_group=?, phone=?, address=? WHERE id = ? -- WHERE id = ? ensures only the selected donor is updated
🗑️ DELETING CUSTOMER — DELETE Donor
DELETE FROM donors WHERE id = ? -- Permanently removes the record from the database
📋 DISPLAYING ALL CUSTOMERS — SELECT All Donors (JTable)
SELECT id, name, age, blood_group, phone, address, donation_date FROM donors ORDER BY id DESC -- ORDER BY id DESC = newest records appear first
🔎 FILTERING — By Blood Type
SELECT blood_group, available_units FROM blood_inventory WHERE available_units < 5 ORDER BY blood_group ASC -- Filters blood types with CRITICAL stock (less than 5 units)
📊 REPORTS — Monthly Donation Trends (GROUP BY)
SELECT MONTHNAME(donation_date) as month, YEAR(donation_date) as year, COUNT(*) as total_donations FROM donors GROUP BY YEAR(donation_date), MONTH(donation_date) ORDER BY year DESC, month DESC
➕ AUTO INVENTORY UPDATE — After Donor Registration
UPDATE blood_inventory SET available_units = available_units + 1 WHERE blood_group = ? -- Automatically adds 1 unit to inventory when a new donor is registered
